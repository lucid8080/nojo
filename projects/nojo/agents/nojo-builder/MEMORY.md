# nojo-builder — MEMORY (long-term notes)

## Store

- Execution patterns that improved outcomes (repeatable checklists)
- Implementation decisions and rationale (link to `projects/nojo/DECISIONS.md` when appropriate)
- Artifact references (what was produced; where it should be consumed)

## Do not store here

- Personality/identity drift rules (those belong in `IDENTITY.md` / `SOUL.md`)
- Other agents’ long-term memory
- Private user personal memory

## Daily notes

Write incremental updates into `memory/<YYYY-MM-DD>.md`.
