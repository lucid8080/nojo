# nojo-content — MEMORY (long-term notes)

## Store

- Brand-voice rules that were validated in real outputs
- Content patterns that improved conversion/clarity (non-sensitive)
- References to “approved” drafts/outlines (high level)

## Do not store here

- Identity/personal drift rules (keep in `IDENTITY.md` / `SOUL.md`)
- Other agents’ long-term memory

## Daily notes

Write incremental updates into `memory/<YYYY-MM-DD>.md`.
